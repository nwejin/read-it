# read-it 프로젝트 초기 세팅

## 서비스 개요

- 서비스명: 읽었나?
- 목적: 집에 있는 책 관리 (보유 여부, 독서 상태 추적)
- 타겟: 가족용 (부모님 포함, 사용성 중요)

## 기술 스택

- Framework: Next.js (TypeScript)
- DB: Supabase
- 책 데이터: 알라딘 Open API
- 배포: Vercel
- 스타일링: Tailwind CSS

## 핵심 기능 (MVP)

1. 알라딘 API로 책 검색
2. 책장에 추가 (보유 여부)
3. 독서 상태 관리 (읽음 / 읽는 중 / 안읽음)

## 알라딘 API

- 검색: GET http://www.aladin.co.kr/ttb/api/ItemSearch.aspx
- 조회: GET http://www.aladin.co.kr/ttb/api/ItemLookUp.aspx
- output=js 로 JSON 받기
- Next.js Route Handler에서 서버사이드 호출 필요 (http라서)
- TTB Key는 환경변수로 관리

## 우선 해줘

1. Next.js + Tailwind CSS 프로젝트 초기 세팅 (폴더 구조 포함)
2. Supabase 테이블 설계 및 연결
3. 모바일 최적화 웹앱 구성 (PWA 고려)

## 버전 업데이트 방법

### 메인 버전 (v1.X) — 새 기능 배포 시

1. **이 파일(CLAUDE.md)** — 로드맵 표에 새 행 추가, 상태 `✅ 완료`
2. **`src/lib/changelog.ts`** — `CHANGELOG` 배열 맨 앞에 새 항목 추가

```ts
// 예시: v1.6 배포 시
{
  version: '1.6',
  date: '2026-xx-xx',
  items: ['새 기능 설명'],
},
```

### 패치 버전 (v1.X1, v1.X2, ...) — UX 개선 등 소규모 업데이트 시

메인 버전 배포 없이 작은 개선을 할 때는 부모 버전 아래에 패치 번호를 붙인다.

1. **이 파일(CLAUDE.md)** — 로드맵 표에 패치 버전 행 추가 (예: v1.51)
2. **`src/lib/changelog.ts`** — 부모 버전 `items` 배열에 날짜와 함께 항목 추가 (별도 버전 항목 X)

```ts
// 예시: v1.5 배포 후 UX 개선 시 (v1.51)
{
  version: '1.5',
  date: '2026-xx-xx',
  items: [
    '기존 기능 설명',
    '개선 내용 (2026-xx-xx)', // 날짜 병기
  ],
},
```

`LATEST_VERSION`은 `CHANGELOG[0].version`을 자동 참조하므로 별도 수정 불필요.
